## Day 1: Hardware Setup & Repository Initialization

Objective: Prepare the physical USB drive and set up the version control safely.

🛠️ Tasks Completed Today:

[x] Repository Setup: Initialized the GitHub repository and established the project roadmap in the README.md.

[x] Safety First (.gitignore): Created a strict .gitignore file. Why? Because portable Python environments (/env/) and local AI models (.gguf) are massive (several gigabytes). If Git tries to track them, the repository will crash. Only the core logic and scripts will be tracked here.

[x] Hardware Prep: Formatted a high-speed USB 3.2 drive to NTFS / exFAT. This is mandatory because FAT32 cannot hold individual files larger than 4GB, and our AI model will be around 4.5GB.

[x] Speed Test: Verified the USB drive read speeds (aiming for ~300 MB/s). High read speeds are critical so the AI "Brain" loads into the computer's RAM quickly when plugged in.

## Day 2: The Portable Environment (/env)

**Objective:** Create a 100% isolated and standalone Python environment directly on the USB drive so the AI doesn't rely on the host computer.

### 🛠️ Tasks Completed Today:
- [x] **Downloaded WinPython:** Acquired a standalone version of Python (WinPython 3.10.x "Dot" version) which is designed to run directly from a portable drive without installation.
- [x] **Extracted to USB:** Carefully extracted the entire WinPython package directly into the root of the USB drive, placing it inside a folder named `/env`.
- [x] **Located the Executable:** Navigated through the new `/env` directory to find the exact path to `python.exe`. This is a critical step because our future launch script needs to know exactly where the "Brain's Engine" is located. *(Typical path: `env\WPy64-31090\python-3.10.9.amd64\python.exe`)*
- [x] **Verified Safety Protocol:** Confirmed that the `.gitignore` file created on Day 1 is successfully blocking this massive 1GB+ `/env` folder from uploading to GitHub, keeping the repository fast and clean.
